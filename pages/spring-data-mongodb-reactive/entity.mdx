## Entity

MongoDB 내의 컬렉션 내에 존재하는 하나의 Document 에 해당하는 객체를 표현하는 클래스를 Entity 라고 부르며, `@Document` 애노테이션을 통해 Entity 를 식별합니다. spring data mongodb reactive 에서는 Document 를 Entity 단위로 식별하기 위해서 필요한 데이터베이스관련 세팅들에 대한 metadata 들을 `@Id`, `@Field` 등과 같은 애노테이션으로 제공해주고 있습니다.<br/>

<br/>





## @Document

클래스 레벨의 애노테이션입니다.<br/>

Entity 로 식별할 데이터의 단위를 표현한 클래스를  `@Document` 애노테이션을 붙여서 적용합니다.<br/>

collection 명을 별도로 지정 가능합니다.<br/>

## @Id

필드 레벨의 애노테이션입니다.<br/>

`_id` 필드에 대해 적용하는 애노테이션입니다.<br/>

Mongodb 의 모든 도큐먼트는 `_id` 필드가 필요합니다.<br/>

MappingMongoConverter 는 아래의 방식을 통해 `_id` 를 식별해서 매핑하거나 자동으로 추가합니다.

- @Id 가 붙어있는 필드가 있을 경우 `_id` 필드를 매핑
- 필드 명이 id 이고 @Field 를 통해서 별도의 이름이 부여되지 않은 경우 이 필드를 `_id` 필드로 매핑합니다.
- id 필드가 제공되지 않는 경우에는 자동으로 `_id` 필드가 추가됩니다.

<br/>



## @DBRef

필드 레벨의 애노테이션입니다.<br/>

MongoDB 의 DBRef 형태로 저장해야 하는 필드에 사용하는 애노테이션 입니다.

## @Indexed

필드 레벨의 애노테이션입니다.<br/>

이 애노테이션이 표시된 필드에 대해서 인덱스를 생성합니다. 기본적으로는 자동 생성이 비활성화이며, 별도의 설정이 필요합니다.

## @CompoundIndex

클래스 레벨의 애노테이션입니다.<br/>

여러 필드로 구성된 복합 인덱스를 적용하고자 할 때 사용하는 애노테이션입니다.<br/>

## @TextIndexed

필드 레벨의 애노테이션입니다.<br/>

필드에 text index 를 적용합니다.<br/>

## @HashIndexed

필드 레벨의 애노테이션입니다.<br/>

필드에 hash index 를 적용합니다.<br/>

<br/>

## @Field

필드 레벨의 애노테이션입니다.<br/>

Entity 로 인식할 클래스 내의 필드에 대해 적용합니다. @Field 가 붙은 필드는 convention 기반이 아닌 Field 에 명시한 name 을 적용해서 인식합니다.<br/>

## @Transient

필드 레벨의 애노테이션입니다.<br/>

Entity 로 인식할 클래스 내의 필드 중 Document 매핑에서 제외할 필드에 대해 @Transient 애노테이션을 적용합니다.<br/>

## @Version

필드 레벨의 애노테이션입니다.<br/>

낙관적 락(Optimistic Lock) 을 적용하고자 할 때 사용하는 애노테이션입니다. Entity 가 Update 될 때마다 자동으로 Update 됩니다.<br/>
