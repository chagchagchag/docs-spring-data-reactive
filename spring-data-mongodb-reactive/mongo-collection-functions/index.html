countDocuments (...)

참고

  • MongoCollection.java (opens in a new tab)

  • countDocuments (opens in a new tab)(ClientSession (opens in a new tab) clientSession, Bson (opens in a new tab) filter, CountOptions (opens in a new tab) options)

Collection 메타 데이터를 이용해서 컬렉션 내의 Document 의 갯수를 반환합니다.

package com.mongodb.client;
@ThreadSafe
public interface MongoCollection<TDocument> {
    /** 
     * Counts the number of documents in the collection according to the given options.
     * ...
     */
    Publisher<Long> countDocuments(ClientSession clientSession, Bson filter, CountOptions options);
}
  • ClientSession (opens in a new tab) : ClientSession 을 통해서 multi document transaction 을 지원합니다.
  • Bson (opens in a new tab) filter : Bson 구현채를 이용해서 filter 연산이 가능합니다.
  • CountOptions (opens in a new tab) options : hint, limit, skip, maxTime, collation 등과 같은 정보를 Option 으로 넘기는 것이 가능합니다.

aggregate (...)

참고

  • MongoCollection.java (opens in a new tab)
  • aggregate (opens in a new tab)(List (opens in a new tab)<? extends Bson (opens in a new tab)> pipeline)
  • Aggregates (opens in a new tab)

pipline 에 해당하는 Bson 식들에 대해 필터링, 집계, 그루핑 등과 같은 aggregation 을 수행합니다.

package com.mongodb.client;
@ThreadSafe
public interface MongoCollection<TDocument> {
    /** 
     * Aggregates documents according to the specified aggregation pipeline.
     * ...
     */
    AggregatePublisher<TDocument> aggregate(List<? extends Bson> pipeline);
}

Aggregates (opens in a new tab) 클래스에서는 aggregation 표현식을 Bson 객체로 생성해내는 다양한 종류의 정적 팩토리 메서드들이 헬퍼 메서드로 존재합니다.

지원되는 메서드 들은 굉장히 다양하며 그 중 대표적으로 잘 알려진 메서드 들은 아래와 같습니다.

  • addFields, set, bucket, bucketAuto
  • sort, sortByCount
  • count
  • match, project
  • skip, limit, lookup
  • facet, graphLookup, group, unionWith, unwind, out, merge, replaceRoot, replaceWith, sample

bulkWrite (...)

참고

  • MongoCollection.java (opens in a new tab)
  • bulkWrite (opens in a new tab)(ClientSession (opens in a new tab) clientSession, List (opens in a new tab)<? extends WriteModel (opens in a new tab)<? extends TDocument (opens in a new tab)>> requests, BulkWriteOptions (opens in a new tab) options)
  • BulkWriteOptions (opens in a new tab)
  • WriteModel (opens in a new tab)

insert, updates, replaces, delete 연산을 한번에 모아서 bulk 연산을 수행합니다.

package com.mongodb.client;
@ThreadSafe
public interface MongoCollection<TDocument> {
    /** 
     * Executes a mix of inserts, updates, replaces, and deletes.
     * ...
     */
    Publisher<BulkWriteResult> bulkWrite(ClientSession clientSession, List<? extends WriteModel<? extends TDocument>> requests,
                              BulkWriteOptions options);
}

입력인자를 보면 WriteModel (opens in a new tab) 이 보입니다. 이 WriteModel 은 interface 이며 구체타입 들은 아래와 같습니다.

  • DeleteManyModel (opens in a new tab) : 조건을 만족하는 모든 Document 를 찾아서 모두 삭제할 수 있도록 표현된 Model 객체입니다.
  • DeleteOneModel (opens in a new tab) : 조건을 만족하는 하나의 Document 를 찾아서 삭제할 수 있도록 표현된 Model 객체입니다.
  • InsertOneModel (opens in a new tab) : 하나의 Document 를 Inert 할 수 있도록 표현된 Model 객체입니다.
  • ReplaceOneModel (opens in a new tab) : 조건을 만족하는 하나의 Document 를 찾아서 Replace 할 수 있도록 표현된 Model 객체입니다.
  • UpdateManyModel (opens in a new tab) : 조건을 만족하는 2개 이상의 Document 를 찾아서 수정할 수 있도록 표현된 Model 객체입니다.
  • UpdateOneModel (opens in a new tab) : 조건을 만족하는 하나의 Document 를 찾아서 수정할 수 있도록 표현된 Model 객체입니다.

package com.mongodb.client;

@ThreadSafe
public interface MongoCollection<TDocument> {

     * Inserts the provided document. If the document is missing an identifier, the driver should generate one.

 * The options to apply to an operation that inserts a single document into a collection.
 */
public final class InsertOneOptions {
    private Boolean bypassDocumentValidation;
    private BsonValue comment;
    // ... 
}
